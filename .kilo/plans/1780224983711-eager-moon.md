# Plan: Fix M-Pesa Automation Gateway Configuration Error

## Root Cause

The error `{"error":"M-Pesa Automation Gateway is not configured by the platform owner."}` is thrown by `app/api/billing/stk-push/route.js:28-29`.

```js
const gConf = await kvGet('paav_global_config', {}, 'platform-master');
const gw = gConf.mpesaGateway;
if (!gw || !gw.consumerKey || !gw.shortcode) {
  return NextResponse.json({ error: 'M-Pesa Automation Gateway is not configured by the platform owner.' }, { status: 400 });
}
```

This endpoint **only** reads M-Pesa credentials from the **platform-level global config** (`paav_global_config.mpesaGateway`), which must be manually set via the Super Admin panel. It does **not** fall back to environment variables.

Meanwhile, the school-level STK flow (`app/api/mpesa/stk/route.js`) already falls back to `process.env.MPESA_*` when school/paybill credentials are missing. The billing endpoint lacks this fallback.

## Fix

**File:** `app/api/billing/stk-push/route.js`

Modify the gateway credential lookup to fall back to environment variables when the global config `mpesaGateway` is absent or incomplete. This aligns with how the regular STK push flow works and lets the user test with already-configured env vars without needing to populate the Super Admin panel first.

### Changes

1. After reading `gConf.mpesaGateway`, if it is missing/incomplete, construct a gateway object from env vars:
   - `MPESA_CONSUMER_KEY`
   - `MPESA_CONSUMER_SECRET`
   - `MPESA_SHORTCODE`
   - `MPESA_PASSKEY`
   - `MPESA_ENV` (default `sandbox`)

2. Keep the existing check, but make it a soft warning or remove it entirely in favor of the env-variable fallback. The safest approach: build `gw` from env vars when the DB config is empty.

### Pseudocode change

```js
const gConf = await kvGet('paav_global_config', {}, 'platform-master');
let gw = gConf.mpesaGateway;

// Fallback to platform-level env vars if global config is not set
if (!gw || !gw.consumerKey || !gw.shortcode) {
  gw = {
    consumerKey:    process.env.MPESA_CONSUMER_KEY    || '',
    consumerSecret: process.env.MPESA_CONSUMER_SECRET || '',
    shortcode:      process.env.MPESA_SHORTCODE      || '',
    passkey:        process.env.MPESA_PASSKEY        || '',
    env:            process.env.MPESA_ENV            || 'sandbox',
  };
}

if (!gw.consumerKey || !gw.shortcode) {
  return NextResponse.json({ error: 'M-Pesa Automation Gateway is not configured.' }, { status: 400 });
}
```
